## 知識檢索模組優化 MVP 開發規劃

---

### 階段一：RAG 檢索基礎建置

**階段目標**  
建立穩定的 RAG 檢索流程，支援向量資料庫與基本關鍵字查詢。


### 1. 建立 Chroma/FAISS 向量資料庫
- 安裝 Chroma 或 FAISS 相關套件
- 設計資料結構，整理旅遊知識文件
- 將文件分割為合適的 chunk，並儲存於向量資料庫

### 2. 整合 OpenAI Embeddings 進行文本向量化
- 設定 OpenAI API Key
- 使用 OpenAI Embeddings 將 chunk 轉換為向量
- 將向量存入 Chroma/FAISS 資料庫

### 3. 實作基本關鍵字檢索功能
- 設計關鍵字查詢介面
- 實作關鍵字比對與文件檢索邏輯
- 測試關鍵字檢索結果

### 4. 測試 RAG 檢索流程正確性與穩定性
- 編寫單元測試，驗證向量與關鍵字檢索結果
- 測試多種查詢情境，確保流程穩定
- 修正發現的問題，優化檢索效能


**Checklist**
- [ ] 建立 Chroma/FAISS 向量資料庫
- [ ] 整合 OpenAI Embeddings 進行文本向量化
- [ ] 實作基本關鍵字檢索功能
- [ ] 測試 RAG 檢索流程正確性與穩定性
